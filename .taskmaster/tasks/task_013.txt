# Task ID: 13
# Title: Implement Multi-Project/Repo Routing
# Status: pending
# Dependencies: 2, 6
# Priority: low
# Description: Extend the configuration layer to support routing Linear events to different GitHub repositories based on rules.
# Details:
Read routing rules from an environment variable (e.g., `ROUTING_RULES_JSON`). The rules should map a Linear team ID or label to a specific `owner/repo` pair. The event router must use these rules to select the correct GitHub target.

# Test Strategy:
Configure multiple routing rules. Send Linear events from different teams and verify that GitHub issues are created in the correct corresponding repositories.
